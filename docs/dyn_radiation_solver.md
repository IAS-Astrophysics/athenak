# Dynamical Radiation Solver

This document records the current public contract of the standalone
`dyn_radiation` solver, the supported runtime modes, and the diagnostic tests
that should be rerun after solver changes.  The longer derivation lives in
`dyngr_radiation_paper/dyngr_radiation_method.tex`; this file is the practical
developer reference.

## Solver Modes

`dyn_radiation` has two geometry modes.

| Mode | Input | Evolved intensity | Intended use |
| --- | --- | --- | --- |
| CKS compatibility | `dyn_radiation/geometry=cks` | legacy \(k^0 k_0 I\) convention | Regression against the existing stationary Cartesian Kerr-Schild `radiation` solver. |
| ADM | `dyn_radiation/geometry=adm` | Eulerian densitized intensity \(\sqrt{\gamma} I\) | Passive radiation transport on ADM/Z4c or analytic ADM backgrounds. |

The ADM mode is metric-passive: radiation sees the ADM/Z4c metric and can
exchange energy-momentum with the fluid, but radiation stress-energy is not fed
back into the Z4c spacetime right-hand side.

Important runtime flags:

| Flag | Meaning |
| --- | --- |
| `dyn_radiation/angular_fluxes` | Enables angular geodesic advection. This is required for curved ADM backgrounds. |
| `dyn_radiation/rad_source` | Enables radiation-fluid absorption/scattering/emission coupling. |
| `dyn_radiation/fixed_fluid` | Holds fluid evolution fixed. Defaults `affect_fluid=false`; explicitly requesting `fixed_fluid=true` and `affect_fluid=true` is rejected. |
| `dyn_radiation/source_max_iter` | Maximum nonlinear source iterations for temperature-dependent opacities. |
| `dyn_radiation/source_tolerance` | Relative/absolute source iteration tolerance. |

## Numerical Updates

The spatial update is finite-volume in position and angle.  In ADM mode, the
cell-centered variable is \(U=\sqrt{\gamma}I\), the coordinate speed is
\(\alpha s^i-\beta^i\), and the geometric energy source is
\[
  G = \alpha K_{ij}s^is^j - s^i\partial_i\alpha .
\]
The current update applies this source with the closed-form multiplicative
increment \(I^{n+1}=I^n\exp(\Delta t\,G)\) within the stage, avoiding the old
forward-Euler positivity failure.

Negative angular bins are repaired by a conservative angular redistribution.
The limiter preserves the local angular zeroth moment: in ADM mode this is the
cell integral of \(\sqrt{\gamma}I\), while in CKS mode it is the legacy primitive
intensity integral.  Production diagnostics should still monitor limiter
activations because frequent activations indicate under-resolved angular or
spatial structure.

The matter source step is iterated for temperature-dependent opacities.  Each
iteration solves the quartic gas-temperature equation with a bracketed
Newton-bisection root finder, recomputes opacities and comoving intensities, and
stops when the gas temperature and intensity updates satisfy
`source_tolerance` or the `source_max_iter` cap is reached.

## Geometry Cache

`Rad_PrepareGeom` rebuilds the tetrad and angular drift cache at the start of
each radiation stage.  For Z4c runs this cache is stage-locked to the ADM data
available at the beginning of the stage, matching the stage data used by the
metric and matter right-hand-side tasks.  `Rad_Newdt` currently rebuilds the ADM
geometry before the timestep reduction as an implementation detail; this is
redundant with the stage geometry task and should be treated cautiously if the
task graph is reordered.

The ADM tetrad path builds:

| Cache | Purpose |
| --- | --- |
| `tet_c`, `tetcov_c`, `norm_to_tet` | Eulerian orthonormal tetrad, inverse tetrad, and normal-frame transforms. |
| `sqrt_detg_c`, `sqrt_detg_x*f` | Cell/face volume normalization for ADM intensity and fluxes. |
| `adm_*_c` | Cell-centered ADM fields used by angular drift and diagnostics. |
| `na` | Angular face speeds generated by the Hamiltonian geodesic operator. |

The sequential Kokkos kernels in this path are ordered by AthenaK's default
execution-space launch ordering.  Keep explicit fences or single-stream
semantics in mind before enabling multi-stream backends.

## Known Limits

- Radiation backreaction on spacetime is intentionally absent.
- The ADM angular operator uses cached second-order metric derivatives.
- The null-particle pusher is diagnostic only; it uses nearest-cell ADM fields.
- Runtime validation has been performed on the local CPU/MPI build. CUDA/HIP/SYCL
  compatibility has been statically reviewed but not runtime-certified in this
  pass.
